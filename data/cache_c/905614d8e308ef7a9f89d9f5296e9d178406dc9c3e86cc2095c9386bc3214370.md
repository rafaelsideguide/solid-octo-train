# Webhooks

The Webhooks API provides a reliable way to manage webhooks operations at scale. This guide covers the core concepts, authentication, and common usage patterns.

## Overview

Webhooks is handled via standard HTTP headers. All requests must be authenticated using a valid API key passed in the `Authorization` header.

## Quick Start

```bash
curl -X POST https://api.example.com/v1/webhooks \
  -H 'Authorization: Bearer YOUR_API_KEY' \
  -H 'Content-Type: application/json' \
  -d '{"key": "value"}'
```

## Error Handling

All errors return standard HTTP status codes. Check the `error` field in the response body for a machine-readable error code and human-readable message.
